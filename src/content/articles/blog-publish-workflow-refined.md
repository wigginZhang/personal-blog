# **博客半自动化发布：我的后台工作流打磨完成**

大家好！今天想和大家分享一下我最近完成的一个小项目——**博客半自动化发布后台工作流**。这个项目让我在开发过程中体验到了很多乐趣，也让我对未来的博客发布有了全新的期待。

## **项目概述**

项目地址：[https://wigginzhang.github.io/personal-blog/](https://wigginzhang.github.io/personal-blog/)

### **前端部分：AI的魔法**

首先，前台部分其实没什么特别的亮点。**前端是由AI生成的**，并托管在GitHub上。技术选型上，我采用了**Astro**，这也是AI推荐的方案。设计风格上，我选择了**渐变亮色**，同样也是AI的建议。不得不说，AI在设计和技术选型上的表现让我非常满意。

### **开发方式：Brainstorm式开发**

这个项目的开发方式是我最喜欢的部分——**Brainstorm式开发**。这种方式让我在开发过程中充满了灵感和创意，就像在和AI一起头脑风暴一样。这种开发体验让我爱不释手。

## **后台工作流：轻量化的CLI工具**

### **传统后台页面的替代方案**

与传统的后台页面不同，我选择了一种更轻量化的方式——**CLI工具**。这种方式让我在终端中就能完成大部分操作，简洁而高效。

### **操作流程**

1. **复制文案**：在终端中复制我想要发布的博客文案。
2. **AI优化**：AI会自动优化文案，并生成博客页面。
3. **配图处理**：如果需要配图，我只需将图片粘贴到指定位置，AI会处理剩下的事情。

这种操作方式让我有一种回到三十年前程序员的感觉，**简朴而美好**。就像我想象中的程序员生活一样，充满了动手的乐趣和创造的满足感。

### **代码示例**

以下是一个简单的代码示例，展示了我在终端中如何使用CLI工具：

```bash
# 复制文案到剪贴板
echo "这是我的新博客文章" | pbcopy

# 使用CLI工具生成博客页面
blog-cli generate --title "新文章标题" --content "$(pbpaste)"

# 如果需要配图
blog-cli add-image --path "/path/to/image.jpg"
```
