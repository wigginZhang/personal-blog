# **Harness 编程体验与 Minimax 的惊喜**

最近，我体验了一把 **Harness 编程**，不得不说，这感觉实在是太棒了！整个过程让我感到一切都在掌控之中，AI 的随机性被有效约束，整个流程井然有序。

## **从讨论到编码：一切尽在计划中**

首先，我们通过讨论确定了方案，**Plan before coding**（编码前先计划）这一原则被严格执行。这样一来，任务分配变得非常清晰，大家都知道自己该做什么。

### **任务分发与工具开发**

在明确任务后，我们迅速进入了开发阶段。我负责开发一个文案润色的小工具，这个工具的主要功能是帮我将博客文章分发出去。令人欣喜的是，这个工具从 P2 到 P3 的过程实现了半自动化。虽然我仍然需要手动复制文本，但我觉得这已经足够了。毕竟，如果完全自动化，可能反而会失去一些灵活性。

## **关于微信平台的思考**

在开发过程中，我意识到一个问题：腾讯微信的 API 似乎是锁死的。作为一个拥有庞大用户基础的平台，微信无疑是许多开发者的首选，但它的封闭性也让人感到无奈。我不禁感叹，**微信就像一座护城河**，不太可能对外开放其 API。

## **Harness 与 Minimax 的完美结合**

在使用 Harness 编程的过程中，我还发现了一个小惊喜。Harness 的 AI 在编码计划方面表现出色，但在文案润色方面却显得有些力不从心。于是，我决定尝试一下 Minimax，没想到这成了我这次体验中的一大亮点。

### **充值的意外收获**

我原本打算充值 10 元来使用 Minimax 的服务，但没想到，Minimax 慷慨地赠送了我 15 元。这让我感到非常惊喜！更令人兴奋的是，润色一次文案的费用仅为 2 分钱。按照这个价格，我觉得这个服务可以用上 10 年以上。

```markdown
**润色费用计算：**
- 每次润色费用：0.02 元
- 赠送金额：15 元
- 可用次数：15 / 0.02 = 750 次
```

## **总结与展望**

总的来说，这次 Harness 编程和 Minimax 的体验让我感到非常满意。以下是我的一些总结：

- **Harness 编程** 提供了强大的可控性和清晰的流程管理。
- **Minimax** 提供了高性价比的文案润色服务，性价比极高。
- **微信平台的封闭性** 让人感到无奈，但也提醒我们寻找更多开放平台的可能性。

未来，我希望能继续探索更多类似的服务，进一步提升我的开发效率和内容质量。如果你也对 Harness 或 Minimax 感兴趣，不妨尝试一下，相信你也会像我一样，爱上这种高效而愉悦的编程体验。
