# 博客发布流程优化 - 图片粘贴集成

**日期：** 2026-04-20
**项目：** personal-blog
**状态：** 设计中

## 背景

当前 `polish-article.ts` 在润色后手动询问是否发布。图片功能已实现但未集成到润色流程。需要优化发布交互，支持在润色过程中无缝插入图片。

## 需求

在润色流程中集成图片粘贴功能，使用【图】作为占位符标记。

## 交互流程

```
1. 输入文字 → 回车两次结束
2. AI 润色
3. 显示润色结果（显示【图】标记位置提示）
4. 询问："是否粘贴图片？(y/n)"
   - n → 跳过，继续步骤 5
   - y → 进入图片粘贴流程：
     a. 显示文章中【图】的位置列表
     b. 提示："请 paste 图片（自动按顺序替换【图】，完成后按 done）"
     c. 每次 paste → 保存图片，替换对应【图】，复制链接
     d. 用户输入 "done" 结束
5. 确认/修改/取消 (y/m/n)
6. 保存 md 文件
7. 询问发布 (y/n)
   - y → git add/commit/push
   - n → 完成
```

## 图片位置处理

| 情况 | 处理 |
|-----|------|
| 【图】存在 | 替换为 `![image](/personal-blog/images/xxx.png)` |
| 无【图】标记 | 默认追加到文章末尾 |

## 实现要点

### polish-article.ts 修改

1. 润色完成后，检测内容中是否有【图】
2. 如果有，提示【图】的数量和位置
3. 询问是否粘贴图片
4. 如果是，进入交互式 paste 循环：
   - 读取剪贴板
   - 保存图片
   - 替换对应【图】
   - 复制链接到剪贴板
   - 继续直到用户输入 done
5. 显示最终内容供确认

### 图片链接格式

`![image](/personal-blog/images/{filename})`
