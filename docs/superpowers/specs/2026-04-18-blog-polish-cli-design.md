# Blog Article Polishing CLI Tool Design Specification

**Date:** 2026-04-18
**Project:** Personal Blog CLI Tool
**Purpose:** Polish raw text into blog-ready Markdown articles

---

## 1. Concept & Vision

A CLI tool that transforms rough notes, drafts, or thoughts into polished personal blog articles. The tool acts as a writing assistant—taking unstructured text and reshaping it into readable, engaging blog posts with appropriate structure, formatting, and Markdown syntax. The human remains in control through an interactive review step before any publishing occurs.

---

## 2. Functionality

### Core Features

1. **Text Input**
   - Accept multi-line text input from terminal
   - Input ends with empty line (double Enter)
   - Support pasting large blocks of text

2. **AI Polishing**
   - Call LLM API to transform text into blog-ready format
   - Style: Personal essay (first-person, conversational, natural)
   - Add appropriate headings, structure the content
   - Apply Markdown formatting (bold, italic, lists, code blocks)
   - Preserve code snippets if present
   - Suggest filename based on content

3. **Interactive Review**
   - Display polished result in terminal
   - Options:
     - **Accept (y)** — Proceed to save and optionally publish
     - **Modify (m)** — Enter modification instructions, re-polish
     - **Cancel (n)** — Discard and exit

4. **File Generation**
   - Generate `.md` file in `src/content/articles/`
   - Auto-generate filename from title
   - Add frontmatter (title, description, date, tags)

5. **GitHub Publishing**
   - After accepting, ask: "Publish to GitHub? (y/n)"
   - If yes: git add → git commit → git push
   - Commit message: `docs: add article - {filename}`

### User Flow

```
1. Run: npm run polish
2. Paste/type raw text
3. Press Enter twice to finish input
4. See polished result
5. Choose:
   - y: Save file
   - m: Enter modification instructions → Go to step 4
   - n: Exit
6. If saved: "Publish to GitHub? (y/n)"
7. If yes: Git add/commit/push
8. Done!
```

## 3. Technical Approach

### File Structure

```
~/projects/personal-blog/
├── scripts/
│   └── polish-article.ts    # Main CLI tool
├── src/
│   └── content/
│       └── articles/        # Generated articles go here
└── package.json             # Add polish script
```

### Dependencies

- **llamacloud-api** or **openai** — LLM API calls
- **simple-git** — Git operations
- **tsx** — Run TypeScript directly

### Environment Variables

- `OPENAI_API_KEY` or `ANTHROPIC_API_KEY` — LLM authentication

### Polishing Prompt

```
You are a blog writer. Transform the user's raw text into a polished personal blog article.

Rules:
- Write in first person, conversational tone
- Add appropriate headings to structure the content
- Use Markdown formatting (bold, italic, lists)
- Preserve any code blocks as-is
- Keep the original meaning and key points
- Add small transitions if content feels disjointed
- Remove obvious grammar mistakes but don't over-edit

Output format:
1. A suggested filename (slug) for the article
2. The polished article in Markdown

Respond with:
FILENAME: <slug>
---
<polished markdown content>
```

### Frontmatter Template

```yaml
---
title: "<Generated Title>"
description: "<Auto-generated from first paragraph>"
date: <current date in YYYY-MM-DD>
tags: []
---
```

## 4. CLI Interface

### Commands

```bash
# Run with npm script
npm run polish

# Or directly
npx tsx scripts/polish-article.ts

# With custom API key
OPENAI_API_KEY=sk-xxx npm run polish
```

### Terminal Interaction

```
=== Blog Article Polisher ===

Paste your raw text below.
Press Enter twice (empty line) when done.

---
[User pastes text]
---

✍️  Polishing with AI...

[Spinner while waiting]

=== Polished Result ===

[Markdown content displayed]

---
Accept this? (y)es / (m)odify / (n)o: y

📁 Saved to: src/content/articles/my-article-slug.md

🚀 Publish to GitHub? (y/n): y

🔄 Pushing to GitHub...
✓ Done! Article published at https://wigginzhang.github.io/personal-blog/
```

### Error Handling

- No API key → Show error with instructions to set environment variable
- API call fails → Show error, offer retry
- Git push fails → Show error, file is saved locally
